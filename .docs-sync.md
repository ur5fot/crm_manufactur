# Documentation Sync Checklist

When making changes to the project, ensure all documentation files are updated:

## Files to Keep in Sync

1. **README.md** (English) - Main user documentation
2. **README.ru.md** (Russian) - Русская документация
3. **CLAUDE.md** - Technical architecture guide

## What to Update When...

### Adding New Features
- [ ] Update feature list in all README files
- [ ] Add technical details to CLAUDE.md
- [ ] Update API endpoints section if applicable
- [ ] Update data model section if schema changed

### Changing Architecture
- [ ] Update CLAUDE.md with architectural changes
- [ ] Update README files if it affects user experience
- [ ] Update project structure diagrams

### Modifying Data Model
- [ ] Update "Data Model" section in all files
- [ ] Update CSV format notes if delimiter/encoding changed
- [ ] Add new dictionary types if applicable

### Changing Setup Process
- [ ] Update "Quick Start" / "Быстрый старт" sections
- [ ] Update "Development Commands" in CLAUDE.md
- [ ] Test the instructions on a fresh install

## Quick Sync Command

```bash
# Compare differences between files
diff README.md README.ru.md
```

## Translation Notes

- Keep technical terms consistent (CSV, API, endpoint)
- Preserve code examples and paths exactly
- Maintain same section structure in all files
